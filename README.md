# 今天吃啥

一个面向选择困难人群的菜谱推荐前端项目。项目从 0 到 1 搭建，当前版本是 MVP 第一版，重点实现“随机推荐菜谱、查看做法、收藏菜谱”这些基础功能。

## 项目简介

“今天吃啥”用于帮助用户快速决定今天做什么菜。用户打开首页后，可以看到 3 道今日推荐菜品，也可以点击“换一换”重新生成推荐。

项目第一版只做前端，不接入后端。菜谱数据保存在本地 `recipes.js` 中，收藏数据使用浏览器 `localStorage` 保存。

## 目标用户

- 每天需要自己做饭的人
- 不知道今天吃什么的人
- 刚开始学习做饭的新手
- 想快速查看家常菜做法的人
- 希望收藏常做菜谱的人

## 核心功能

- 首页随机推荐 3 道菜
- 点击“换一换”重新推荐菜品
- 尽量避免连续两次推荐上一轮已经展示过的菜
- 菜谱卡片展示图片、菜名、用时、难度、口味
- 点击“查看做法”进入菜谱详情页
- 详情页展示食材、步骤、小贴士等信息
- 支持收藏和取消收藏菜谱
- 我的收藏页展示已收藏菜谱
- 收藏数据保存在浏览器本地
- 移动端、平板、电脑端响应式布局

## 技术栈

- JavaScript

- localStorage

## 本地运行方法

```bash

```

启动开发服务器：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

启动后在浏览器访问：

```text
http://127.0.0.1:5173/
```

## 当前版本说明

当前版本是 MVP 第一版，主要目标是验证基础产品流程：

1. 用户进入首页。
2. 系统随机推荐 3 道菜。
3. 用户可以换一组推荐。
4. 用户可以查看菜谱详情。
5. 用户可以收藏喜欢的菜谱。
6. 用户可以在收藏页查看和取消收藏。

目前项目没有后端，所有数据都在前端本地处理。

## 项目亮点

- 从 0 到 1 完成一个可运行的 React 前端项目
- 使用 React Router 实现首页、详情页、收藏页的页面切换
- 使用本地数据模拟真实业务数据，适合 MVP 阶段快速验证
- 使用 localStorage 实现收藏功能，不依赖后端
- 抽象 `RecipeCard` 组件，在首页和收藏页复用
- 将收藏相关逻辑集中在 `storage.js`，避免组件重复操作存储
- 使用原生 CSS 实现移动端优先的响应式布局
- 项目结构清晰，适合作为前端学习和简历展示项目
