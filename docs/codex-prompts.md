# 今天吃啥 Codex 提示词记录

## 本项目用过的 Codex 指令

下面是本项目开发过程中使用过的主要指令类型，整理后方便后续继续复用。

### 1. 初始化项目结构

```text
请帮我初始化“今天吃啥”项目的基础结构。
使用 React + Vite，JavaScript，不使用 TypeScript。
创建 pages、components、data、utils 目录。
每个页面先写简单占位内容。
```

适合用于从空项目开始搭建基础目录。

### 2. 配置 React Router

```text
请帮我为“今天吃啥”项目配置 React Router。
首页路径：/
菜谱详情页路径：/recipe/:id
我的收藏页路径：/favorites
BottomNav 提供首页和我的收藏入口。
```

适合用于添加页面路由和基础导航。

### 3. 创建本地菜谱数据

```text
请创建 src/data/recipes.js 文件。
导出 recipes 数组，至少包含 20 道中国家常菜。
每道菜包含 id、name、image、time、difficulty、servings、taste、category、ingredients、steps、tips。
内容适合厨房新手。
```

适合用于补充本地演示数据。

### 4. 开发首页推荐功能

```text
请继续开发 Home 首页，实现“今日推荐菜品”功能。
从 recipes.js 导入数据。
页面加载时随机推荐 3 道菜。
添加“换一换”按钮，点击后重新随机推荐。
尽量避免连续两次推荐完全相同。
```

适合用于实现首页核心推荐逻辑。

### 5. 开发菜谱卡片跳转

```text
请完善 RecipeCard 组件。
显示图片、菜名、用时、难度、口味。
添加“查看做法”按钮。
点击后跳转到 /recipe/:id。
```

适合用于把列表卡片和详情页连接起来。

### 6. 开发菜谱详情页

```text
请开发 RecipeDetail 菜谱详情页。
使用 useParams 获取 id。
从 recipes.js 中找到对应菜谱。
展示菜名、图片、用时、难度、适合人数、口味、分类、食材、步骤、小贴士。
如果没找到，显示没有找到这个菜谱和返回首页按钮。
```

适合用于实现详情页数据读取和展示。

### 7. 接入收藏功能

```text
请为项目接入收藏功能。
在 RecipeCard 和 RecipeDetail 中添加收藏按钮。
点击收藏按钮可以添加或取消收藏。
收藏数据使用 src/utils/storage.js 中的方法。
不要在组件中重复写 localStorage 逻辑。
```

适合用于实现收藏状态和本地存储。

### 8. 开发收藏页

```text
请开发 Favorites 我的收藏页。
从 storage.js 的 getFavorites() 读取收藏 id。
根据收藏 id，从 recipes.js 中找到对应菜谱。
复用 RecipeCard 展示收藏列表。
取消收藏后页面列表立即更新。
没有收藏时显示空状态。
```

适合用于开发收藏列表页。

### 9. 优化 UI 和移动端

```text
请优化“今天吃啥”项目的 UI 样式和移动端体验。
设计目标：简洁、温和、清爽。
卡片式布局，图片比例统一，按钮样式统一。
底部导航固定在底部。
手机一列，平板两列，电脑三列。
使用原生 CSS，不引入 UI 框架。
```

适合用于统一视觉风格和响应式布局。

## 后续开发可以复用的提示词

### 增加分类筛选

```text
请为首页增加分类筛选功能。
分类来自 recipes.js 中的 category 字段。
用户点击分类后，只在该分类下随机推荐菜谱。
如果分类下菜谱不足 3 道，可以少展示或允许重复。
代码保持简单，适合初学者阅读。
```

### 增加搜索功能

```text
请为首页增加搜索功能。
用户可以输入菜名或食材关键词。
搜索结果复用 RecipeCard 展示。
如果没有搜索结果，显示“没有找到相关菜谱”。
不要引入额外搜索库。
```

### 增加口味筛选

```text
请为首页增加口味筛选功能。
口味来自 recipes.js 中的 taste 字段。
筛选后推荐结果只显示对应口味的菜。
保持逻辑简单，不要影响收藏页和详情页。
```

### 优化收藏页

```text
请优化 Favorites 收藏页。
增加“清空收藏”按钮。
点击后清空 localStorage 中的收藏 id。
清空后显示空状态。
需要二次确认，避免误点。
```

### 增加 README

```text
请为项目创建 README.md。
内容包括项目简介、技术栈、如何安装依赖、如何启动开发服务器、主要功能和目录结构。
要求中文，适合新手阅读。
```
