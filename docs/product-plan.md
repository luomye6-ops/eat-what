# 今天吃啥产品计划

## 项目背景

“今天吃啥”是一个帮助用户快速决定吃什么的菜谱推荐软件。很多人在做饭前会反复纠结：不知道吃什么、不知道家里现有食材能做什么、也不想看太复杂的菜谱。

本项目第一版先做 React 前端，不做后端。菜谱数据放在本地 `src/data/recipes.js`，收藏数据保存在浏览器的 `localStorage` 中。

## 目标用户

- 每天需要自己做饭的人
- 不知道今天吃什么的人
- 刚开始学做饭的厨房新手
- 想快速找到家常菜做法的人
- 希望收藏常做菜谱的人

## 用户痛点

- 菜谱太多，越看越难选
- 很多菜谱步骤复杂，不适合新手
- 临时做饭时，只想快速得到几个推荐
- 常做的菜没有地方快速保存
- 手机端看菜谱时，信息排版不清楚

## 第一版功能

第一版目标是做一个简单可用的前端原型，重点是“能推荐、能查看、能收藏”。

已规划并实现的核心功能：

- 首页随机推荐 3 道菜
- 点击“换一换”重新推荐菜品
- 尽量避免连续两次推荐完全相同的菜
- 菜谱卡片展示图片、菜名、用时、难度、口味
- 点击“查看做法”进入菜谱详情页
- 详情页展示食材、步骤、小贴士等完整信息
- 支持收藏和取消收藏菜谱
- 我的收藏页展示已收藏菜谱
- 收藏数据保存在浏览器本地
- 移动端、平板、电脑端都有基础适配

## 后续迭代方向

后续可以按从简单到复杂的顺序继续迭代：

1. 增加分类筛选，例如家常菜、素菜、汤、主食、快手菜。
2. 增加口味筛选，例如清淡、酸甜、微辣、咸鲜。
3. 增加搜索功能，可以按菜名或食材搜索。
4. 增加“不吃什么”选项，例如不吃辣、不吃肉、不吃豆腐。
5. 增加根据人数推荐用量的功能。
6. 增加更多真实菜谱图片。
7. 增加一周菜单功能，帮助用户提前规划。
8. 后续如果需要多人同步收藏，可以再考虑接入后端和数据库。
